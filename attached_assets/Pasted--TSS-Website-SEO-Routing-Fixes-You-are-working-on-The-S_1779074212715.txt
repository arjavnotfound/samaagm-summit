## Fixes to Make

### 1. Fix direct URL routing on Netlify
When someone opens `/event` (or any route) directly via a link, they get a Netlify 404 instead of the React app. This is because Netlify looks for an actual file and finds nothing.

Fix: create a `public/_redirects` file with:
```
/* /index.html 200
```

This tells Netlify to always serve the React app and let React Router handle routing.

### 3. Add `sitemap.xml` to `public/`

### 5. Update `index.html` — meta description + canonical tag
Two small changes to `<head>` in `index.html`:

**a) Update the meta description** to include "Indore" — right now it never mentions the city, which hurts local search. Something like:
```
India's first democratic youth summit in Indore — where students, thinkers, and future leaders come together to debate, build, and lead.
```
Update this in all three places it appears (meta description, og:description, twitter:description).

**b) Add a canonical tag** right after the `<title>`:
```html
<link rel="canonical" href="https://thesamaagmsummit.netlify.app/" />
```

### 6. Add structured data (JSON-LD) to `index.html`

<script type="application/ld+json">

  "organizer": {
    "@type": "Organization",
    "name": "The Samaagm Summit",
    "url": "https://thesamaagmsummit.netlify.app"
